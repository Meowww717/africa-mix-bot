# 🌍 Африканці — Telegram Bot

Бот для організації міксових тренувань у групі. Учасники записуються самостійно або парою (чоловік + жінка), адмін керує зустрічами та ігровим розкладом через особисті повідомлення.

---

## 📁 Файли

| Файл              | Призначення                                |
| ----------------- | ------------------------------------------ |
| `bot.py`          | Основний бот (aiogram)                     |
| `import_users.py` | Перегляд учасників групи (telethon)        |
| `bot.db`          | SQLite база даних (зберігається на volume) |

---

## ⚙️ Запуск

### 1. Встановити залежності

```bash
pip install aiogram telethon
```

### 2. Встановити змінні середовища (fly.io secrets)

```bash
fly secrets set TOKEN=ваш_токен_від_botfather
fly secrets set ADMIN_ID=ваш_telegram_id
fly secrets set TELEGRAM_API_ID=ваш_api_id
fly secrets set TELEGRAM_API_HASH=ваш_api_hash
```

### 3. Задеплоїти на fly.io

```bash
fly deploy
```

## 📋 Команди

### Адмін

| Команда         | Опис                                      |
| --------------- | ----------------------------------------- |
| `/create`       | Створити нову зустріч (обрати день і час) |
| `/manage_users` | Відкрити панель управління учасниками     |

> `/create` викликається в груповому чаті — вибір дня і часу приходить адміну в особисті, зустріч публікується в чат. Після створення адмін отримує дубль зустрічі в особисті з адмінськими кнопками.

### Панель `/manage_users`

| Кнопка               | Дія                                                 |
| -------------------- | --------------------------------------------------- |
| ➕ Додати учасника   | Вручну: `user_id, ім'я, прізвище, username, gender` |
| ❌ Видалити учасника | Видалити юзера за `user_id`                         |
| 📋 Показати всіх     | Список учасників з позначкою статі (ч)/(ж)          |

## 🏐 Кнопки зустрічі (публічні — видно всім)

| Кнопка             | Хто може                 | Дія                                                         |
| ------------------ | ------------------------ | ----------------------------------------------------------- |
| 🏐 Записатись      | Учасник з бази           | Сам(а) / З партнером з чату / З гостем                      |
| 👫 Додати партнера | Записаний одиночно       | Список вільних з бази + одиночні що вже записались + вручну |
| 👤 Додати гостя    | Будь-який учасник з бази | Самостійно або парою, з вибором статі                       |
| ❌ Скасувати запис | Учасник з бази           | Тільки себе або себе і партнера                             |

## 🔧 Адмінські кнопки (тільки в особистих адміна)

Після створення зустрічі адмін отримує дубль повідомлення в особисті. Це повідомлення оновлюється автоматично коли хтось записується або скасовує запис.

| Кнопка                | Дія                                                               |
| --------------------- | ----------------------------------------------------------------- |
| 🔀 Розбити по парах   | Рандомно об'єднує одиночних чоловіків і жінок у пари              |
| 📋 Розписати ігри     | Питає кількість кортів, публікує розклад на 7 раундів в групу     |
| 🔀 Сайдаут-розподіл   | Рандомно ділить пари по кортах (можна тиснути багато разів)       |
| 🔄 Міксанути пари     | Перемішує всіх у нові пари М+Ж, публікує 2 раунди в групу         |
| 🪑 Хто сидить першим? | З'являється тільки якщо рівно 3 пари — рандомно обирає хто сидить |
| 🗑 Видалити зустріч   | Видаляє зустріч з групи і всіх учасників                          |

---

## 📝 Як записатись

1. Натисни 🏐 Записатись в груповому чаті
2. Бот напише в особисті — обери варіант:
   - **Сам(а)** — одиночний запис
   - **З партнером з чату** — список людей протилежної статі: вільні з бази, вже записані одиночні, або ввести ім'я вручну
   - **З гостем** — вводиш ім'я, гість автоматично іншої статі
3. Якщо записався одиночно і хочеш додати партнера пізніше — кнопка 👫 Додати партнера

## 🎮 Ігровий розклад

### 📋 Розписати ігри

Адмін обирає 1 або 2 корти — бот генерує 7 раундів по 20 хв (~2.5 год) і публікує в групу.

| Умова                | Логіка                                             |
| -------------------- | -------------------------------------------------- |
| 2 пари               | Розклад не потрібен                                |
| 3 пари               | Використай 🪑 Хто сидить першим?                   |
| 3 пари + 1 одиночний | 1 корт, сайдаути, одиночний грає з тим хто сидить  |
| 4+ пари, 1 корт      | Сайдаути — по 3 пари грають, решта сидять по черзі |
| 4+ пари, 2 корти     | Рівномірно по кортах, пари міняються щороунду      |

### 🔀 Сайдаут-розподіл

Рандомно ділить всі пари по 2 кортах. Можна тиснути скільки завгодно разів для нового розподілу.

### 🔄 Міксанути пари

Перемішує всіх учасників у нові пари М+Ж (гарантовано різні від оригінальних). Публікує 2 раунди в групу.

## 🗄 База даних

### `users`

| Поле         | Опис                |
| ------------ | ------------------- |
| `user_id`    | Telegram ID         |
| `first_name` | Ім'я                |
| `last_name`  | Прізвище            |
| `username`   | @username           |
| `gender`     | `male` або `female` |

### `meetings`

| Поле               | Опис                                     |
| ------------------ | ---------------------------------------- |
| `meeting_id`       | Унікальний ID зустрічі                   |
| `message_id`       | ID повідомлення в груповому чаті         |
| `chat_id`          | ID групового чату                        |
| `day`              | День тижня                               |
| `time`             | Час                                      |
| `creator_id`       | Telegram ID адміна який створив зустріч  |
| `admin_message_id` | ID дублю повідомлення в особистих адміна |

### `participants`

| Поле           | Опис                         |
| -------------- | ---------------------------- |
| `meeting_id`   | ID зустрічі                  |
| `user_id`      | ID юзера (від'ємний = гість) |
| `display_name` | Ім'я для відображення        |
| `pair_id`      | ID пари (NULL = одиночний)   |
| `gender`       | `male` або `female`          |

---

## ⚠️ Важливо

- База даних зберігається на fly.io volume `/data/bot.db` — не зникає після деплою
- При першому деплої створи volume: `fly volumes create bot_data --region fra --size 1 --yes`
- Якщо переходиш зі старої версії бази — виконай через `fly ssh console`:

```bash
python3 -c "
import sqlite3
conn = sqlite3.connect('/data/bot.db')
conn.execute('ALTER TABLE meetings ADD COLUMN creator_id INTEGER')
conn.execute('ALTER TABLE meetings ADD COLUMN admin_message_id INTEGER')
conn.commit()
"
```

- Всі адмінські дії відбуваються в особистих — груповий чат залишається чистим
- Пари завжди чоловік + жінка — бот не дозволяє створити пару однієї статі
